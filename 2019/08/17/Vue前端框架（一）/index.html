1.1vue是什么？

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

其实说简单点vue就是一个单页面应用程序(single page application)，简称spa。是对ajax异步加载技术的极致使用。例如网易云音乐、coding

这是一款非常优秀的前端javascript框架，由尤雨溪创建开发，可以轻松构建“SPA应用程序”，通过“指令”扩展了HTML，通过“表达式”绑定数据到HTML，最大限度上解放了DOM

1.2网站交互方式

  • 经典的多页面(京东，淘宝)
  • 前后端糅合在一起，开发效率低下
  • 用户体验一般，点击刷新跳转，等待时间过长
    • 每个页面都需要重新加载渲染，速度慢
    • 有利于SEO搜索引擎搜索（蜘蛛会爬链接）
  • 现代式的单页面
    • 前后端分离，开发效率高，可维护性好
      • 服务端不关心页面，只关心通过接口拿数据和服务端交互，处理页面
      • 客户端不关心数据库及数据操作，只关心通过接口拿到的数据
    • 用户体验好，就像一个 原生的客户端软件一样使用
    • 只要加载渲染局部视图即可，不需要整个页面加载
    • 单页面应用开发技术复杂，不是特别成熟，所以诞生了很多开发框架
      • AngularjS
      • ReactjS
      • VuejS
    • 单页面由于是异步加载过来的，所以不会被搜索搜索引擎搜索到，不利于SEO
    • 无法兼容低版本浏览器

1.3前后端分离开发模式

  • 项目立项
  • 需求分析
  • 服务端的工作
    • 需求分析
    • 设计数据库
    • 接口设计（有时候也需要前端参与其中）
    • 接口(处理数据)开发
  • 前端的工作
    • 需求分析
    • 写页面
    • 通过接口湖服务端进行交互

v-if指令

v-if 指令用于条件性地渲染一块内容。这块内容只会在指令的表达式返回 true值的时候被渲染

也可以用 v-else 添加一个“else 块”,v-else 元素必须紧跟在带 v-if 或者 v-else-if 的元素的后面，否则它将不会被识别。

在

因为 v-if 是一个指令，所以必须将它添加到一个元素上。但是如果想切换多个元素呢？此时可以把一个 <template> 元素当做不可见的包裹元素，并在上面使用 v-if。最终的渲染结果将不包含 <template> 元素。

v-show指令

v-show指令的用法也是根据条件展示，不同的是v-show的元素始终会渲染于dom中，而v-show只是简单地切换了元素的display属性而已，注意，v-show 不支持 <template> 元素，也不支持 v-else。

v-if和v-show的区别

v-if 是“真正”的条件渲染，因为它会确保在切换过程中条件块内的事件监听器和子组件适当地被销毁和重建。

v-if 也是惰性的：如果在初始渲染时条件为假，则什么也不做——直到条件第一次变为真时，才会开始渲染条件块。

相比之下，v-show 就简单得多——不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换。

一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。
