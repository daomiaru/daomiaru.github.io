git和github

操作Git的基本工作流程就是先修改文件，然后执行”git add”命令。”git add”命令会把文件加入到暂存区，这样就可以执行”git commit”命令，将文件存入文档库，从而形成一次历史纪录

分支

一开始的默认主分支为master,为了避免多人协作时代码提交有误给他人或自己带来的影响。这里我们可以基于master分支创建一个分支，默认新创建的分支会拥有主分支的所有内容。这时候我们就可以对该新分支做修改而不会影响主分支的内容了。

github基本使用

1.通过git clone 命令下载远程仓库到本地

2.修改完成后通过git命令放到本地仓库

3.通过git push命令提交到远程仓库进行代码托管

git开源项目协作工作流程

1.fork一个项目

2.clone项目到本地

3.在本地修改完之后，push项目到自己fork过来的仓库中。

4.pull request提交请求给对方

5.对方审查通过没有问题之后选择是否合并你的改动

6.对方合并你的改动

7.流程结束
