4.transiton和animation的区别

transtion和animation大部分属性时相同的，它们都是随时间改变元素的的属性值，它们的主要区别时transition需要触发一个时间才能改变属性，而animation不需要触发任何时间。并且transition为两帧，而animation可以做到不止两帧

5.flex布局（弹性布局）

Flex 是 Flexible Box 的缩写，意为”弹性布局”，用来为盒状模型提供最大的灵活性。 布局的传统解决方案，基于盒状模型，依赖 display 属性 + position 属性 + float 属性。它 对于那些特殊布局非常不方便，比如，垂直居中就不容易实现。

PS：主轴就是弹性盒子子元素沿着排列的轴，与主轴垂直的轴称为侧轴

常用属性：

  • flex-direction(适用于父容器上)：决定子item的排列方式（即决定主轴的方向）
    • row：左对齐横向排列（默认）
    • row-reverse：右对齐横向排列
    • column：纵向排列（自上而下）
    • column-reverse：纵向排列（自下而上）
  • flex-wrap(适用于父容器上)：决定item超出父容器是否换行
    • wrap：当子元素溢出父容器换行
    • nowrap：当子元素溢出父容器时不换行
    • wrap-reverse：当子元素溢出父容器时换行，方向和wrapx相反
  • flex-flow(复合属性)：可同时设置flez-direction和flex-wrap
  • justify-conten(适用于父容器上)：设置子元素在主轴上的对齐方式
    • flex-start：盒子元素向行起始位置对齐
    • flex-end：盒子元素向行结束位置对齐
    • centter：向行中间位置对齐
    • space-between：平均分布在行里
    • space-around：平均分布在行里，并在两端保留子元素与子元素之间间距大小的一半
  • align-items(适用于父容器上)：设置子元素在侧轴上的对齐方式
    • flex-start：侧轴起始处开始
    • flex-end：侧轴结束位置
    • center：侧重中间对齐
  • a;ien-self(适用于单独的子元素)：设置单独子元素的alien-items对齐方式。默认为auto,表示继承父元素的alien-item
  • order(适用于子元素)：设置子元素的排列顺序，数值小的排在前面，可以为负值
  • flex-basis：固定px
  • flex-grow(适用于子元素)：根据扩展因子比率分配空间
  • flex-shirk(适用于子元素)：根据收缩因子比率来分配空间

10.visibility=hidden, opacity=0，display:none

  • opacity=0：该元素隐藏起来了，但不会改变页面布局，并且，如果该元素已经绑定一些事件。比如说click,那么点击该区域，也能触发该事件
  • visibility=hidden：该元素隐藏起来了，但不会改变页面布局，也不会触发已经绑定的事件
  • display=none：把元素隐藏起来了，并且会改变页面布局，可以理解为在页面中把该元素删除了一样

11.双边距重叠问题（外边距折叠）

多个相邻普通流的块元素垂直方向magin会重叠

折叠的结果为：

  • 两个相邻的外边距都是正数时，折叠结果是它们之间较大的值
  • 两个相邻的外边距都是负数时，折叠结果是两者绝对值的较大值
  • 两个外边距一正一负时，折叠结果是两者相加的和

12.position属性

  • 固定定位fixed：元素的位置相对于浏览器窗口是固定位置，即使窗口是滚动的它也不会移动。fixed定位是元素的位置与文档流无关，因此不占据空间。fixed定位ide元素和其他元素重叠
  • 相对定位relative：如果对一个元素进行相对定位，它会相对于其正常位置进行定位，使用相对定位时无论是否进行移动，元素仍然占据原来的空间，因此，移动元素会导致它覆盖其他框
  • 绝对定位absolute：绝对定位的元素的位置相对于其最近的非static定位的父元素。若没有已定位的父元素，则其相对于htm页面进行定位。absolute定位的元素于文档流无关，因此不占据空间，absolute定位的元素可以与其他元素重叠
  • 默认定位static：没有定位，元素出现在正常流中

15.CSS选择器有哪些，优先级呢？

id选择器，class选择器，标签选择器，伪元素选择器，伪类选择器

  • 同一元素引用了多个样式时，排在后面的样式属性的优先级高
  • 根据选择器的类型不同，优先顺序为：id选择器>class选择器>标签选择器
  • 标签之间存在层级包含关系时，后代元素会继承祖先元素的样式。如果后代元素定义了与祖先元素相同的样式，则祖先元素的相同样式属性会被覆盖，继承的样式的优先级比较低，至少比标签选择器的优先级低
  • 带有!important标记的样式属性的优先级最高
  • 样式表的来源不同时，优先级顺序为：内联样式>内部样式>外部样式>浏览器用户自定义样式>浏览器默认样式

29.说一说网格布局grid

flex布局是轴线布局，只能指定针对轴线的位置，可以看作是一维布局，grid布局则是将容器划分成“行”，“列”，产生单元格，然后指定“项目所在”的单元格，可以看作是二维布局。grid布局远比flex布局强大

3.单元格

33.讲讲什么是BFC他能解决什么问题

前端程序员在布局时,经常会遇到很多问题,比如当我们定义一个盒子时,理论上盒子里面的元素应该不会影响到盒子外面的元素。但是实际结果并不是我们想的那样

从理论上来说被包含在父元素中的元素是不会影响到父元素外的元素，但实际上并不是如此。那么有没有什么方法将子元素与父元素外部的元素隔离开来呢？

答案就是使用BFC(block formatting context) “块级格式化上下文”,它的目的就是形成一个完全独立的空间，让空间里的子元素不会影响到外面的布局

我们通过为元素设置一些CSS属性，就能触发这一空间

  • float不设为none
  • position不为relative和static（脱离文档流）
  • overflow为hidden、auto和scroll
  • dispaly的值为table-cell和inline-block

解决问题

第一种：解决浮动元素高度坍塌的问题

假设一个父元素中有几个子元素，子元素设置float，则父元素的高度会坍塌。这是因为子元素脱离了文档流，导致父元素无法检测到它们的存在导致高度坍塌了，后面的布局也会乱掉。

这个时候我们就可以给父元素设置BFC，触发了BFC的容器就是页面上的一个完全隔离开的容器，容器中的元素绝对不会影响到容器外面的元素，为了保证这个规则，触发了BFC的父元素在计算高度时，不得不让浮动的子元素也参与进来

  • overflow:hidden
  • display:table-cell 、block 、inline-block
  • position:fixed、absolute
  • float:flet 这样会让父元素也脱离文档流，优点时代码量少，缺点是这是一种将错就错的方法，会影响之后的元素排列
第二种：解决自适应布局的问题

两栏布局：左侧定宽，右侧自适应，利用块级元素会尽量占满一行的效果，保证右侧自适应

  • 左边float:left，width:200px

    右边：margin-left:200px

  • 左边绝对定位，top：0，left：0，boottom：0，width:200px

    右边：margin-left：200px

第三种：解决外边距垂直方向重合问题
