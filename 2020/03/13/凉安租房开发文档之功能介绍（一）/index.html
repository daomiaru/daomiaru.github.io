传统的身份验证

JWT的登录验证

使用基于 Token 的身份验证方法，在服务端不需要存储用户的登录记录。大概的流程是这样的：
1、客户端使用用户名跟密码请求登录
2、 服务端收到请求，去验证用户名与密码
3、验证成功后，服务端会签发一个 Token，再把这个 Token 发送给客户端
4、客户端收到 Token 以后可以把它存储起来，比如放在 Cookie 里或者 Local Storage 里
5、客户端每次向服务端请求资源的时候需要带着服务端签发的 Token
6、服务端收到请求，然后去验证客户端请求里面带着的 Token，如果验证成功，就向客户端返回请求的数据

我们这里采用的是jwt方式的登录验证，详细的教程可以自行百度学习，这里不多赘述。

重点来了铁子们！！！

通常的app，当我们登录一次之后，app都会将登录信息缓存下来，这样的话我们就不需要每次重启一次app就登录一次。从用户体验的角度上看这非常的有必要。

刚刚我们提到了token,并且登录成功之后将他保存在了本地，当我们重启app时，我们可以读取本地缓存中的token,并将其发送给服务器，服务器根据秘钥对token进行解析，解析成功则表明是同一用户，这时将我们需要的信息返回给我们。并且token在服务端可以设置过期时间，这也就是为什么我们有的时候长时间不登录app,则我们的信息会消失的原因。

当我们重启应用时，在App.vue中的onLaunch（当uni-app 初始化完成时触发（全局只触发一次））方法中有

可以看到，在这里我们先读取了本地的缓存，若之前有登录过app，则登录之后返回的数据应该都保存在userInfo中，读取成功之后我们向服务器请求验证token，这里userid的作用是返回对应user的数据它同样保存在数据库中，表明用户的唯一性。（其实这个地方应该是验证token之后就表明它是唯一的,不过若想要通过token查到对应的数据，这里还需要将token保存在user表中，我这里觉得太麻烦就省略了，改用userid来返回唯一用户数据，token用于持久化验证）

看看/checklogin接口

1.首先通过verify方法验证token,若失败表明token失效了，也就是我们常说的token过期。这时候需要我们重新登录一下来签发新的token(退出登录之后同样需要重新申请token)

2.验证通过之后，我们根据userid来查表，找出对应用户的数据并将其返回给客户端

显然我们拿到了我们想拿到的数据，此时同样提交mutations中的login方法，将vuex中的数据更新

所以我们此时点开个人页，依然可以看到我们之前登录的信息。这就是登录持久化的过程

只要token没有过期或者不退出登录，我们无论重启多少次app,依然可以保持登录状态
